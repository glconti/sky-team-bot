## Key Decisions

1. **Solo Mode is Testing Feature, Not Game Variant** — All 7 module rules unchanged; domain aggregate is mode-agnostic
2. **GameMode as Domain Enum** — TwoPlayer (default) / Solo; enables future rule variation without breaking changes
3. **Seat-Based Turn Tracking** — Application layer's `GetSeatForUser()` naturally supports solo mode (same userId in both seats)
4. **No Domain Turn-Flow Changes** — Existing `PlaceDie` / `SwitchPlayer` validation works unchanged in solo mode
5. **Simultaneous Visible Placement** — Player sees all 8 dice (4 Pilot + 4 Copilot) with full visibility; supports testing scenarios
